UPGRADE: Complete redesign of the Wealthly Settings page.

The current Settings page is minimal — only currency dropdown and a flat 
category list with budget percentages. Transform it into a comprehensive, 
well-organized control center that covers EVERY aspect of the user's 
financial app experience: profile, preferences, categories for ALL types 
(income + expenses + investments + assets), notifications, security, 
integrations, data management, and more.

PART 3 — GLOBAL UX PATTERNS
═══════════════════════════════════════════════════════════════════════════════

AUTO-SAVE BEHAVIOR:
- Most changes save automatically on blur/change
- Visual feedback: "Saving..." → "Saved ✓" (fades after 2s)
- Critical changes (password, delete) require explicit confirmation

SEARCH:
- Top of settings page: "Search settings..." input
- Real-time filtering of all settings across all sections
- Search indexes section titles + setting labels + descriptions

BREADCRUMBS:
- On mobile: back arrow + current section title
- On desktop: "Settings / [Section]"

UNDO:
- Toast with "Undo" button appears after destructive actions
- 10-second undo window for things like delete category

VALIDATION:
- Inline validation (red text below invalid field)
- Save buttons disabled until valid
- Helpful error messages, never just "Error"

TOOLTIPS:
- Hover "?" icons next to complex settings
- Contextual help without leaving the page

SMART INSIGHTS (consistent with other pages):
Sprinkle small AI insights throughout:
- "Based on your spending, consider 55/25/20 split instead of 50/30/20" 
  in Categories
- "Users in your region typically enable 2FA" in Security
- "You haven't used 3 integrations — consider removing?" in Integrations

═══════════════════════════════════════════════════════════════════════════════
CRITICAL RULES
═══════════════════════════════════════════════════════════════════════════════

1. Settings must be HIGHLY ORGANIZED. No flat mega-page. The 13 sections 
   with clear sub-nav prevent overwhelm.

2. Categories management is the STAR — it must handle Income, Expenses, 
   AND Investment/Asset types with equal power.

3. Every setting change must have clear visual feedback (saved indicator).

4. Dangerous actions are visually separated in Danger Zone with red 
   accents and require confirmation.

5. Consistent with main app design language — same sidebar, cards, 
   typography, colors, hover states.

6. Accessibility is non-negotiable — all toggles labeled, keyboard 
   navigable, screen reader compatible.

7. Mobile experience should feel native (iOS-style drill-in) not a 
   shrunk desktop.

8. The AI Assistant settings reflect that AI is distributed throughout 
   the app (not a standalone page), so users control it globally from here.

Reference aesthetic: Linear's settings × Notion's settings × Apple's 
System Settings × Stripe Dashboard settings. Organized, powerful, beautiful.
