Redesign the "Budget Planning" page for a wealth management web app called 
"Wealthly". The current page is a passive list of empty category rows with 
zero intelligence. Transform it into an active, visual budgeting command 
center that reads real-time spending data from the Expenses page, shows 
allocated-vs-spent comparisons, and triggers smart overspend warnings — 
while matching the existing dashboard's design system exactly.

═══════════════════════════════════════════════════════════
CORE FUNCTIONALITY (CRITICAL — design must reflect these behaviors)
═══════════════════════════════════════════════════════════

1. Budget page READS from Expenses page in real-time:
   - Every category shows: Budgeted amount vs Actual spending this month
   - Actual spending is pulled automatically from transactions tagged to 
     that category on the Expenses page
   - No manual entry of "spent" — it's always live

2. Three threshold states per category, each with distinct visual treatment:
   - SAFE (< 75% spent): green progress bar, calm
   - WARNING (75-95% spent): amber progress bar, warning icon, subtle alert
   - DANGER / OVER (≥ 95% or exceeded): red progress bar, pulsing alert icon, 
     "⚠️ Over budget by $X" message

3. Cross-page alert system (design the visual language):
   - When a category hits WARNING or DANGER state on Budget page, an alert 
     banner appears on the Expenses page for that category
   - Show in the mockup: a notification badge on sidebar "Expenses" item 
     showing count of warned categories

4. Income-driven allocation:
   - Top of page shows total income available to budget
   - As user sets limits, remaining "unallocated" income updates live
   - Visual feedback when over-allocating (exceeding income)
